# Security Policy 🛡️

We take the security of our platform, repositories, and community members very seriously. If you believe you have found a security vulnerability in the JKOSI Registry, please read this document to learn how to report it responsibly.

---

## Supported Versions

Only the latest version of the repository on the `main` branch is supported for security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | ✅ Yes             |
| < 1.0   | ❌ No              |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

If you discover a security issue, please report it privately:
1. Email your findings directly to [security@jkosi.org](mailto:security@jkosi.org).
2. Include a detailed description of the vulnerability, steps to reproduce, and any potential exploits or proof-of-concept code.
3. We will acknowledge receipt of your email within 48 hours and work with you to resolve the issue promptly.

---

## Disclosure Policy

We follow a responsible disclosure policy:
* We request that you give us a reasonable amount of time to fix the reported vulnerability before publishing details publicly.
* We will keep you updated on our progress as we work on a security patch.
* Once resolved, we will publish a security advisory and attribute credit to you for the discovery (if desired).

Thank you for helping keep the JKOSI community safe! 🛡️
